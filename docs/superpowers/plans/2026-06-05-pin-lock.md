# PIN Lock Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Gate all four pages of the OHS-SF static site behind a PIN code (`797283`), remembered for 30 days via `localStorage`, with a branded `lock.html` entry page.

**Architecture:** An inline synchronous `<script>` added to the `<head>` of each HTML page checks `localStorage` before the browser paints anything; if unauthenticated it redirects to `lock.html`. `lock.html` is a standalone page using the existing `styles.css` design tokens that collects the PIN, validates it client-side, writes a 30-day expiry token, and redirects back to the originally requested page.

**Tech Stack:** Vanilla HTML/CSS/JS · localStorage · existing `styles.css` design system

```html
<body>
<div class="lock-page">
  <div class="lock-card" id="lockCard">
    <a class="lock-brand" href="index.html" aria-label="Open Health Stack Foundation home">
      <span class="logo-mark" aria-hidden="true"><i></i><i></i><i></i><i></i></span>
      <span class="brandname"><b>Open Health Stack</b><span>Software Foundation</span></span>
    </a>
    <div class="lock-eyebrow">Private preview</div>
    <h1 class="lock-heading">Enter access code</h1>
    <form id="lockForm" novalidate>
      <input id="pinInput" class="lock-input" type="password" inputmode="numeric"
             maxlength="10" autocomplete="off" placeholder="······" aria-label="Access code" />
      <button type="submit" class="btn btn-primary lock-btn">Continue</button>
    </form>
    <div class="lock-error" id="lockError" role="alert" aria-live="polite"></div>
  </div>
</div>
<script>
(function () {
  var CORRECT_PIN = '797283';
  var EXPIRY_MS = 30 * 24 * 60 * 60 * 1000;

  function getReturn() {
    var params = new URLSearchParams(location.search);
    var ret = params.get('return');
    if (!ret) return 'index.html';
    try {
      var url = new URL(ret);
      if (url.origin !== location.origin) return 'index.html';
      return ret;
    } catch (e) { return 'index.html'; }
  }

  var card  = document.getElementById('lockCard');
  var form  = document.getElementById('lockForm');
  var input = document.getElementById('pinInput');
  var error = document.getElementById('lockError');

  form.addEventListener('submit', function (e) {
    e.preventDefault();
    if (input.value.trim() === CORRECT_PIN) {
      localStorage.setItem('ohs_auth', JSON.stringify({ exp: Date.now() + EXPIRY_MS }));
      window.location.replace(getReturn());
    } else {
      input.value = '';
      error.textContent = 'Incorrect code — try again';
      card.classList.remove('shake');
      void card.offsetWidth;
      card.classList.add('shake');
      card.addEventListener('animationend', function h() {
        card.classList.remove('shake');
        card.removeEventListener('animationend', h);
      });
      input.focus();
    }
  });
})();
</script>
</body>
</html>
```

## Task 2: Add Inline Auth Guards to All Four Pages

**Files:**
- Modify: `index.html`
- Modify: `foundation.html`
- Modify: `projects.html`
- Modify: `community.html`

The guard script must be inserted as the **first child of `<head>`**, immediately after `<meta charset="UTF-8" />`. It runs synchronously before any other resource loads, preventing the page from rendering if unauthenticated.

The guard to insert (same in all four files):

```html
<script>
(function(){
  var a = localStorage.getItem('ohs_auth');
  if (a) { try { if (Date.now() < JSON.parse(a).exp) return; } catch(e){} }
  window.location.replace('lock.html?return=' + encodeURIComponent(location.href));
})();
</script>
```
